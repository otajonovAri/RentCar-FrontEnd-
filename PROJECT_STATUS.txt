  B. REQUEST BODY MAYDON NOMLARI NOMUVOFIQLIKLARLI
────────────────────────────────────────────────────────────────────────────────

  1. MASHINA YARATISH — POST /api/cars
     ❌ Hozir:   CreateCarDto.carModelId = 5
     ✅ To'g'ri:  { "modelId": 5 }
     Fayl: src/types/cars.ts  →  carModelId → modelId

  2. MASHINA RASM QO'SHISH — POST /api/cars/{id}/images
     ❌ Hozir:   AddCarImageDto { url, isMain, displayOrder, altText }
     ✅ To'g'ri:  { "imageUrl": "https://...", "isMain": false }
     Fayl: src/types/cars.ts  →  url → imageUrl, displayOrder/altText olib tashlash

  3. CARS FILTER — GET /api/cars
     ❌ Hozir:   CarsFilter.minDailyRate, maxDailyRate
     ✅ To'g'ri:  minPrice, maxPrice
     Fayl: src/types/cars.ts  →  maydon nomlarini o'zgartir

  4. CARS FILTER — modelId etishmayapti
     ❌ Hozir:   CarsFilter'da modelId yo'q
     ✅ To'g'ri:  ?modelId=2 filtri qo'shish
     Fayl: src/types/cars.ts  →  modelId?: number qo'shish

  5. OWNER PAYOUT YARATISH — POST /api/ownerpayouts
     ❌ Hozir:   CreatePayoutDto { rentalId, ownerRevenuePercent, notes }
     ✅ To'g'ri:  { "rentalId": 99 }  (ownerRevenuePercent backend tomonidan hisoblanadi)
     Fayl: src/types/ownerPayouts.ts  →  CreatePayoutDto soddalashtirish

  6. GOOGLE LOGIN — POST /api/auth/google
     ❌ Hozir:   authApi.googleLogin → { "accessToken": "..." }
     ✅ To'g'ri:  { "idToken": "Google ID token" }
     Fayl: src/api/authApi.ts  →  accessToken → idToken

  7. JARIMA STATUS — FineStatus type
     ❌ Hozir:   type FineStatus = 'Pending' | 'Paid' | 'Cancelled'
     ✅ To'g'ri:  'Unpaid' | 'Paid'
     Fayl: src/types/fines.ts  →  FineStatus type o'zgartirish
     ⚠️  Bu o'zgartirish bilan FinesPage status filterini ham yangilash kerak!

  8. OWNER PAYOUT STATUS — OwnerPayoutStatus type
     ❌ Hozir:   'Pending' | 'Processing' | 'Paid' | 'Failed' | 'OnHold'
     ✅ To'g'ri:  'Pending' | 'Paid' | 'Cancelled'
     Fayl: src/types/ownerPayouts.ts  →  OwnerPayoutStatus o'zgartirish
